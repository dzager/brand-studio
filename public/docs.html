Organic Docs.

Everything you need to create on-brand, SEO-optimized, human-sounding content — from a single prompt.

Overview

Organic is an AI content studio that generates publication-ready blog posts and hero images from a single prompt. Unlike generic AI writers, Organic embeds a full brand engine into every generation — ensuring every article sounds like your best writer wrote it and ranks like your SEO team built it.

Brand Engine

Tone, voice, archetypes, guardrails, and editorial guidelines compiled into every prompt automatically.

Multi-Model

Choose from GPT, Claude, and Gemini models for writing. Use separate models for image generation and utilities.

Fact-Check Consul

Cross-verify claims using multiple AI models (Gemini + Grok) for true multi-source validation.

Auto-Humanize

Detects and rewrites 17+ AI writing patterns so content passes detection tools and reads naturally.

Quick Start

  1. Create a Company — Go to the Companies page and set up your brand with a name, tagline, tone, archetype, and target audiences.
  2. Set Up Voice — Paste a sample article into the Voice Analyzer to extract your brand's exact writing profile (tone descriptors, sentence rhythm, vocabulary level, banned phrases).
  3. Write Your First Prompt — From the Studio page, select your company, choose a model, and describe the article you want. Hit Create.
  4. Review & Refine — Use Humanize, Fact-Check, or Deep Consul to polish the output. Refresh images, search for media, or composite product shots.
  5. Download — Export your complete article bundle (HTML, images, SEO metadata, and JSON-LD structured data).

Studio

The Studio is your main workspace for generating content. Select a company, describe the article you want in natural language, and Organic handles the rest — from prompt assembly to image generation.

Creation Controls

  • Company Selector — Required. Determines which brand engine configuration is used for generation.
  • Prompt Templates — Pre-saved prompts attached to companies. Click to load a template, or write a custom prompt from scratch.
  • Image Style — Select from Default or custom image styles defined in your company settings. Organic will recommend a style using AI if you click Recommend.
  • Model — Choose which AI model writes the article. Options include GPT-5.x, Claude Sonnet, and Gemini models.
  • Length — Short (300–500 words), Medium (800–1,200), Long (1,500–2,500), Deep Dive (2,500–4,000), or No Limit.

Preview Prompt

Click Preview Prompt to inspect the exact system, user, and image prompts that will be sent to the model — including estimated token count. You can copy individual sections or all prompts at once.

Check Overlap

Before generating, use Check Overlap to scan your existing article library for semantic similarity. If a closely matching article already exists (above 75% similarity), you'll see a warning with the match percentage and slug — preventing duplicate content.

Companies

Companies are brand configurations that control every aspect of content generation. Each company has its own brand engine, voice profile, image styles, editorial guidelines, and prompt templates.

Company Settings

  • Name & Tagline — Injected into every prompt for brand identity.
  • Mission — Used to align article angle and value proposition.
  • Brand Archetype — Pathfinder, Innovator, Caregiver, Sage, Creator, Hero, Explorer, or Rebel. Shapes the voice personality and decision-making framework of generated content.
  • Tone — Comma-separated tone descriptors (e.g. "confident, clear, modern") applied to all output.
  • Target Audiences — Used for positioning and language calibration.
  • Photography / Image Style — Directs the AI image generation aesthetic.
  • Primary & Secondary Colors — Applied to design tokens and image palette guidance.
  • Avoid Phrases — Words and phrases the brand never uses.
  • Editorial Guidelines — Freeform text block for company-specific voice, citation rules, content depth requirements, and formatting preferences. Overrides base engine rules when in conflict.
  • SEO Content Guidelines — Company-specific SEO rules that supplement the built-in framework.
  • Reference Articles — URLs of gold-standard articles used as style references during generation.
  • Auto-Humanize — When enabled, articles are automatically humanized after generation (~30s extra).
  • Include Table of Contents — Generates a clickable TOC nav element at the top of each article.

Custom Image Styles

Each company can define multiple image style categories with unique narratives, storytelling cues, and prompt directives. Styles can be either AI Prompt (standard image generation) or Composite Blend (product-on-background compositing with built-in background removal).

Extract Style from Image

Upload any reference photo and Organic will use GPT-5.4 vision to reverse-engineer its visual characteristics — color palette, lighting, lens, film quality, mood, composition, depth of field, texture, saturation, and post-processing — into a structured image style category. Review the AI analysis grid, name the style, and save it for future use across all article generations.

Voice & Prompt Actions

Voice analysis and prompt template management are accessible via modal overlays from the company list. Click the microphone icon to open the Voice Analyzer modal, or the document icon to view and manage prompt templates.

Brand Portal

The Brand Portal (/company) provides a dedicated view for team members to explore and edit their company's full brand configuration — identity, visual styles, voice profile, editorial guidelines, and prompt templates.

  • Inline editing — Toggle into edit mode to modify any field directly from the portal. An Edit/Save/Cancel header bar manages form state and persistence.
  • Reference article management — Add and remove gold-standard reference URLs used as style benchmarks during content generation.
  • Image style extraction — Upload reference images directly from the portal to extract and save new visual style categories.
  • Collapsible sections — Voice profile, editorial guidelines, SEO rules, and image styles are organized into expandable panels for clean navigation.

Brand Engine

The Brand Engine is the core system that compiles every company configuration — archetype, tone, voice profile, editorial guidelines, SEO rules, guardrails, and banned phrases — into a structured system prompt. This prompt is injected into every AI call, ensuring consistent, on-brand output.

How it works When you click Create, Organic compiles a comprehensive system prompt that includes: editorial credibility rules, first-sentence quality enforcement, specificity mandates, anti-AI pattern detection, structure and extraction guidelines, SEO targeting, source attribution requirements, freshness signals, structured data guidance, internal linking strategy, and your company's custom editorial and SEO guidelines.

Guardrails

The engine enforces strict guardrails to prevent common AI writing pitfalls:

  • Banned Phrases — 70+ phrases that never appear in output (e.g. "let's dive in," "game-changer," "in today's landscape").
  • Allowed Tone — Confident, clear, modern, helpful, calm.
  • Disallowed Tone — Bureaucratic, cold, condescending, fear-based, hyperbolic.
  • Content Patterns to Avoid — Dense text blocks, overuse of disclaimers, overpromising outcomes, vague generalizations.

Voice Analysis

The Voice Analyzer extracts a detailed writing profile from sample content. Paste any article (text or HTML) and Organic will analyze it to produce a complete voice profile including:

  • Tone Descriptors — How the writing sounds (e.g. "direct, informative, casual").
  • Sentence Rhythm — Average sentence structure and pacing patterns.
  • Paragraph Style — Typical paragraph length and transition patterns.
  • Vocabulary Level — Reading level and word choice tendencies.
  • Rhetorical Devices — Identified techniques the writer uses.
  • Point of View — First-person, second-person, or third-person conventions.
  • Sample Phrases — Characteristic phrases to emulate.
  • Banned Phrases — Specific phrases the brand should never use.
  • Structural Do's & Don'ts — Patterns to replicate and patterns to avoid.
  • Specificity & Length Rules — Precision and depth requirements.

You can also generate a Voice Priority Prompt Template directly from the voice profile — a structured prompt that forces the AI to prioritize voice matching above all other style guidance.

Humanization

The Humanizer is a comprehensive rewriting system based on Wikipedia's "Signs of AI writing" guide. It detects and rewrites 17+ categories of AI artifacts:

  • Undue emphasis on significance and legacy
  • Superficial analyses with -ing endings
  • Promotional, advertisement-like language
  • Vague attributions and weasel words
  • Formulaic "Challenges and Future Prospects" sections
  • Casual dramatic framing & negative editorial coloring
  • Overused AI vocabulary (delve, landscape, pivotal, crucial, etc.)
  • Copula avoidance ("serves as" instead of "is")
  • Negative parallelisms and Rule of Three overuse
  • Elegant variation (synonym cycling)
  • Em dash overuse, filler phrases, excessive hedging
  • Generic positive conclusions
  • Collaborative communication artifacts ("I hope this helps")
Goes beyond removal The Humanizer doesn't just strip AI patterns — it actively injects personality. It varies sentence rhythm, adds opinions where appropriate, acknowledges complexity, and uses first-person when it fits. The goal is writing that has a human behind it.

Fact-Checking

Organic provides two levels of fact-checking:

Quick Check

A single-model fact check that scans the article for claims and evaluates each one as accurate, unverifiable, misleading, or inaccurate. Returns a confidence percentage and suggested edits for flagged claims.

Deep Consul

The Fact-Check Consul sends the article to multiple independent AI models (Gemini and Grok) for independent verification. Each model evaluates every factual claim, and the Consul synthesizes their verdicts into a consensus with an agreement level:

  • Full Agreement — Both models agree on the verdict.
  • Partial Agreement — Models agree on direction but differ on severity.
  • Models Disagree — Contradictory verdicts requiring human review.
  • Single Source — Only one model returned a result.

Each claim shows the individual model reasoning (expandable), linked sources, and a suggested rewrite that you can apply with one click to update the article directly.

Image Generation

Every article generates a hero image automatically. The image prompt is compiled from your company's photography style, color palette, narrative cues, and the selected image style category.

Image Gallery

Generate multiple image variations using the Refresh Image button. Add custom direction (e.g. "more warm lighting" or "include a laptop on the desk") to guide regeneration. All variations are saved in an in-session gallery where you can switch between them.

Web Image Search

Search the web for stock photos, product images, or any visual asset. Results show title, domain, dimensions, and can be:

  • Used — directly as the article's hero image
  • Downloaded — via the image proxy
  • Composited — used as input for the composite engine

YouTube Search

Search YouTube for related videos, tutorials, and media to embed or reference in your content.

Composite Images

The Composite Engine creates professional product-on-background images using an automated pipeline:

  1. Background Acquisition — Uses an AI-generated background from a scene prompt, a searched web image, or a custom URL.
  2. Product Extraction — Sends the product image to Gemini for green-screen replacement, then applies chroma-key transparency with anti-aliased edge smoothing.
  3. Intelligent Sizing — Scales the product cutout to fill ~80% of the frame with sharpening for upscaled images.
  4. Ambient Color Sampling — Samples the background's edge colors and applies a subtle tint to the product for natural color integration.
  5. Shadow Generation — Creates a realistic soft drop shadow from the product's alpha channel.
  6. Final Composite — Layers background → shadow → tinted product using rule-of-thirds positioning.

Companies can define Composite Blend image styles with default background prompts, background images, and product search queries — making compositing a one-click operation during article creation.

Content Architecture

The Articles page provides two views for managing your content library:

Graph View

A visual canvas showing all articles and clusters as an interconnected network. Articles are color-coded by company and connected to their cluster. Click any node to view its details.

Outline View

A two-level sidebar tree (Cluster → Article) with a full detail panel. Each article displays its human-readable title with a compact role indicator badge — P (Pillar), S (Supporting), or L (Long Tail) — with descriptive tooltips on hover. The cluster panel uses a consolidated action bar: a primary "Generate" button plus a single overflow menu for interlinking, export, sharing, and deletion.

Topical Clusters

Organic supports topical cluster strategies for building authority around core topics:

  • AI Cluster — Describe a topic and let AI generate a full cluster strategy with pillar, supporting, and long-tail pages, including internal linking instructions and keyword targets.
  • Manual Cluster — Create a cluster and manually assign existing articles to it.
  • Auto Cluster — Automatically group existing articles into clusters using embedding-based semantic similarity.

When generating articles within a cluster, Organic injects sibling page context — slugs, keywords, and linking instructions — directly into the prompt so each article naturally links to its cluster siblings.

SEO & GEO

Every article is generated with built-in SEO and Generative Engine Optimization (GEO):

  • Keyword Strategy — Primary and secondary keywords derived from the topic with optimized placement in titles, headings, and body.
  • Featured Snippet Targeting — Direct 40–60 word answers formatted for Google snippet extraction.
  • FAQ Schema — Mandatory FAQ section with H3→p structure for FAQPage schema markup.
  • HowTo Schema — Procedural content formatted as ordered lists for HowTo schema extraction.
  • JSON-LD Structured Data — Full structured data exported with every article download (FAQ, HowTo, Article schema).
  • Meta Tags — Click-optimized title (50–60 chars), compelling meta description (150–160 chars), and keyword-rich slug.
  • Source Attribution — Inline hyperlinked citations to authoritative sources (minimum 4–6 per article).
  • Freshness Signals — Year references and "as of" qualifiers for time-sensitive content.
  • Entity Optimization — Key entities wrapped in <strong> tags with full official names on first use.
GEO Differentiation Every article must include original analysis, decision frameworks, statistical breakdowns, edge case coverage, and practitioner-level guidance that cannot be easily synthesized from other sources — making content more likely to be cited by AI answer engines.

Admin Dashboard

A dedicated admin dashboard for platform operators — visible only to users in the platform_admins table. Provides full oversight and management of all accounts and users.

Accounts Tab

  • Account management — Create, edit, and delete accounts. Assign companies and manage team member rosters.
  • Usage dashboard — Article counts, cluster totals, and billing status across all accounts at a glance.
  • Invitation management — View and revoke pending invitations from the admin panel.

Users Tab

  • User directory — Searchable list of all users with email, name, registration date, last sign-in, and email confirmation status.
  • Account associations — Each user row shows linked accounts with role, plan, and account name.
  • User deletion — Admins can remove users directly from the dashboard with confirmation.

Settings

Access Settings from the sidebar gear icon. Configure default AI models for three process categories:

  • Writing Model — The primary model for article generation (default: GPT-5.4).
  • Image Generation — The model used for hero images (default: GPT Image 2).
  • Utility Model — Used for SEO analysis, style recommendations, and other utility tasks (default: GPT-4.1 Mini).

Settings are persisted in your browser's localStorage and applied automatically to all future generations.

MCP Server

Organic exposes your complete brand toolkit as a single, unified Model Context Protocol (MCP) server — an open standard that lets any compatible AI client (Claude, Cursor, ChatGPT, Windsurf, your own apps) access your brand's voice, photography style, and blog generation pipeline directly.

Instead of copy-pasting brand guidelines into every prompt, the MCP server gives AI clients live access to your Supabase-stored brand profiles. Update a voice profile in Organic → every connected AI client sees the change instantly.

MCP Setup

The Organic Brand MCP server is a standalone Node.js process that communicates over stdio. Add it to any MCP-compatible client.

1. Install dependencies

cd mcp/organic-brand-mcp && npm install

2. Claude Desktop

Edit ~/Library/Application Support/Claude/claude_desktop_config.json:

{
  "mcpServers": {
    "organic-brand": {
      "command": "npx",
      "args": ["tsx", "/path/to/mcp/organic-brand-mcp/src/index.ts"],
      "env": {
        "SUPABASE_URL": "https://your-project.supabase.co",
        "SUPABASE_ANON_KEY": "your-anon-key"
      }
    }
  }
}

3. Cursor

Add to .cursor/mcp.json in your project with the same format, using relative paths.

4. Example

Once connected, ask any MCP-compatible AI:

"Write a blog post about dental implants for pacific-dental and save it"

The AI will automatically call the right tools — get_blog_system_prompt, get_voice_profile, get_user_prompt_template to generate the article, then create_article to save it — all from the same server. No need to specify which server to use.

Prompt Templates

The MCP server includes pre-built prompt templates — guided workflows that Claude follows step by step. In Claude Desktop, access them by typing / in the chat input.

PromptParametersWhat it does
write_article company, topic Fetches all brand context (system prompt, voice, editorial, banned words), generates a full article, then offers to fact-check, validate tone, generate a hero image, or save to the library.
brand_review company, text Validates pasted text against the brand's banned phrases, voice profile, and editorial rules. Provides a brand alignment rating (Strong / Moderate / Weak) and offers to rewrite problem areas.
onboard_brand company Step-by-step guided setup for a new brand — checks if it exists, then walks through tagline, archetype, tone, audiences, sample articles, banned words, photography style, and colors one question at a time.

Using prompts in Claude Desktop

  1. Type / in the chat input to see available prompt templates.
  2. Select a prompt (e.g. write_article).
  3. Fill in the parameters (company name, topic, or text to review).
  4. Claude follows the workflow automatically — fetching brand data, generating content, and offering follow-up actions.
