# Video methodology

Brand-consistent video uses the same layered approach as image generation, plus motion, plus a script. The output is either:

- A **video spec** (script + storyboard + per-scene image prompts + motion spec) — handed off to a human or downstream tool
- A **fully rendered video** — when a motion provider is configured

The spec is the source of truth. The render is one consumer of the spec.

## The four artifacts

A `video` command produces all four:

### 1. Script

The text, written in the brand's voice (using VOICE.md). Scenes are numbered. Each scene has:

- **VO** (voiceover): what the narrator says
- **Caption** (optional): on-screen text
- **Duration**: target seconds (typically 3-8 per scene)

Example:

```markdown
## Scene 1 (0:00–0:05)
VO: Marriage-based green cards take 14 to 38 months. Where you file matters.
Caption: 14–38 months. Service center makes the difference.

## Scene 2 (0:05–0:11)
VO: Vermont Service Center: 14 months. Nebraska: 22. Texas: 28. Potomac: 38.
Caption: VSC 14 | NSC 22 | TSC 28 | PSC 38

## Scene 3 (0:11–0:18)
VO: But the form matters more than the center. I-130 alone is faster than I-130 plus I-485 concurrent.
Caption: I-130 vs concurrent: 9 months gap, average
```

### 2. Storyboard

For each scene, a written description of what the viewer sees. Same structure as an image brief — subject, action, environment, lighting, mood. The storyboard is what gets sent to the image generator (one image per scene).

### 3. Per-scene image prompts

Each scene gets a fully composed image prompt using the `image-prompts.md` methodology. The image prompt inherits the brand's photography style (from VISUAL.md) and the chosen image style category.

A scene's image prompt looks like every other image prompt the skill produces — same five layers — with the scene's storyboard text in the scene-brief slot.

### 4. Motion spec

How the static scene image becomes motion. The spec is provider-agnostic; the same spec can drive Sora, Runway, Veo, Luma, or any future motion model.

```yaml
scene: 1
motion_type: dolly_in | pan_left | pan_right | static | tilt_up | tilt_down | parallax | zoom_in | zoom_out
motion_intensity: subtle | moderate | strong
duration_seconds: 5
focal_action: "narrator off-screen explains, on-screen subject is mid-conversation in a warm-lit kitchen"
camera_notes: "handheld feel, slight breathing motion"
audio_notes: "ambient room tone under VO"
```

## The motion vocabulary

Same five-layer image methodology, plus motion. The motion vocabulary stays minimal and physical:

| Motion type | Use when |
|---|---|
| `static` | The subject does the work. Person speaking, product hero. Default for talking-head scenes. |
| `dolly_in` | Tightening focus on the subject. Used for emphasis at a turning point in the script. |
| `dolly_out` | Revealing context. Used at the open of a scene to establish location. |
| `pan_left` / `pan_right` | Following an action or revealing parallel context. |
| `tilt_up` / `tilt_down` | Vertical reveals — typically environments (a building, a tree, a tall figure). |
| `parallax` | Subtle depth motion for documentary feel without committing to a camera move. |
| `zoom_in` / `zoom_out` | Sparingly. Punctuation. |

Don't combine motions in a single scene. One motion per scene. Cuts between scenes carry the variety.

## Motion intensity

| Intensity | Movement amount | Use case |
|---|---|---|
| `subtle` | 5-10% frame movement over the scene | Default. Adds life without distraction |
| `moderate` | 10-25% movement | Emphasis moments, transitions |
| `strong` | 25-50% movement | Rare. Action emphasis, narrative pivots |

Avoid `strong` except for genuine action moments. Most professional documentary editing uses `subtle` throughout.

## Audio (out of scope for v1 motion render)

Audio is part of the spec but generation is downstream. The spec captures:

- **VO**: the narrator's text. The user records it or uses a TTS model with their preferred voice profile.
- **Music**: brand-tied music cues (genre, mood, intensity). Not generated by the skill.
- **Ambient**: location-appropriate room tone.
- **SFX**: action sound effects (if any).

The audio mix is the human's job in v1. The spec describes it; the motion render produces silent footage.

## Provider abstraction

The `video` command emits the spec unconditionally. It optionally calls a motion provider per scene when `IMPRINT_VIDEO_PROVIDER` is set.

| Provider | env var | Default model | Notes |
|---|---|---|---|
| OpenAI Sora | `OPENAI_API_KEY` | sora-2 | Best for cinematic motion |
| Runway | `RUNWAY_API_KEY` | gen-4 | Best for image-to-motion |
| Google Veo | `GOOGLE_API_KEY` | veo-3 | Best for natural motion physics |
| Luma | `LUMA_API_KEY` | dream-machine-2 | Best for stylized motion |
| None | unset | spec only | Default |

The provider abstraction is a thin shim: take the per-scene image + motion spec, call the provider, get a clip back. Stitching is downstream — `video` does not assemble a final cut.

## End-to-end render flow (when a provider is configured)

```
1.  Generate the script from the topic + BRAND.md + VOICE.md
2.  For each scene:
    a. Compose the image prompt (image-prompts.md methodology)
    b. Generate the scene image via IMPRINT_IMAGE_PROVIDER
    c. Compose the motion spec
    d. Call IMPRINT_VIDEO_PROVIDER with the image + motion spec
    e. Receive the clip
3.  Emit: script + storyboard + scene images + scene clips + master spec.json
4.  Hand off to the user for final assembly (audio + edit + color)
```

Final assembly is intentionally out of v1. The motion clips and spec are sufficient for any video editor (Premiere, Final Cut, Resolve, DaVinci) to produce the finished video in 15-30 minutes.

## Spec schema

The complete video spec emitted by `video`:

```yaml
video_spec_version: "1.0"
brand:
  name: "Greencard Law"
  voice_profile: { ...from VOICE.md... }
  visual_style: { ...from VISUAL.md... }
topic: "I-130 service center comparison"
content_type: "explainer"   # explainer | testimonial | product_demo | brand_film | tutorial
duration_target_seconds: 60
scenes:
  - number: 1
    duration: 5
    vo: "Marriage-based green cards take 14 to 38 months."
    caption: "14–38 months"
    storyboard: "Tight on a wall clock in a warm-lit room. Late afternoon natural light from a window. Slight motion blur on the second hand."
    image_prompt: "<full composed image prompt>"
    image_url: "(filled in after generation)"
    motion:
      type: dolly_in
      intensity: subtle
      duration_seconds: 5
      camera_notes: "slow inward dolly toward the clock face"
      audio_notes: "ambient room tone under VO"
    clip_url: "(filled in after render)"
  # ... more scenes
audio:
  vo_recording: "(human-supplied or TTS)"
  music: "warm acoustic, low intensity"
  ambient: "indoor home office"
  sfx: []
delivery:
  aspect_ratio: "16:9"
  resolution: "1920x1080"
  frame_rate: 30
  formats: ["mp4_h264", "mov_prores"]
```

## What NOT to do

- **Don't try to render the final cut from the skill.** Assembly belongs in an editor.
- **Don't combine motion types per scene.** One motion, one scene.
- **Don't use `strong` motion as the default.** Subtle is the default.
- **Don't ignore the brand's photography style for video.** The same VISUAL.md powers every scene image; the brand's visual coherence carries through to motion.
- **Don't pretend a 30-second AI-generated piece is a finished video.** Treat the output as B-roll-quality footage to be assembled, color-graded, audio-mixed by a human.
